# AGENTS.md

Guidance for AI coding agents working in this repository.

## Project Overview

This is a C++ firmware project targeting the ESP32 microcontroller, built with
ESP-IDF and managed via CMake and Conan. All developer workflows are driven by
shell scripts in `scripts/`. Run any script from the repository root.

## Repository Layout

```
components/          Reusable C++ components (one subdirectory per component)
main/                ESP32 app entry point (app_main)
tests/               Host-native unit tests (Conan + CMake + Catch2)
scripts/             Developer workflow scripts (see below)
build/               ESP-IDF build output — do not commit
tests/build/         Native test build output — do not commit
```

## Scripts Reference

### `scripts/setup.sh` — Verify or install prerequisites

```
./scripts/setup.sh            # check only; exits 1 if anything is missing
./scripts/setup.sh --install  # install missing tools, leave existing ones alone
./scripts/setup.sh --update   # upgrade all tools and install any missing
```

Checks: cmake, ninja, python3, conan (via pipx), git, clang-format, clang-tidy,
idf.py (ESP-IDF), qemu-system-xtensa.
Run this first on a new machine.

---

### `scripts/build.sh` — Build ESP32 firmware

```
./scripts/build.sh
```

Activates the ESP-IDF environment automatically if not already sourced, then
runs `idf.py build`. Output is written to `build/`.

---

### `scripts/emulate.sh` — Run firmware in QEMU

```
./scripts/emulate.sh
```

Builds the firmware first if `build/stembot.bin` is absent, then launches
`idf.py qemu monitor`. Press **Ctrl+]** to quit.
Requires `qemu-system-xtensa` (installed by `setup.sh --install`).

---

### `scripts/flash.sh` — Flash firmware to a physical ESP32

```
./scripts/flash.sh
STEMBOT_PORT=/dev/ttyUSB0 ./scripts/flash.sh   # override serial port
```

Default ports: `/dev/ttyUSB0` (Linux), `/dev/tty.usbserial-0001` (macOS).
If you get a permission error on Linux: `sudo usermod -aG dialout $USER` then
re-login.

---

### `scripts/monitor.sh` — Open a serial monitor

```
./scripts/monitor.sh
STEMBOT_PORT=/dev/ttyACM0 ./scripts/monitor.sh  # override serial port
```

Connects to the attached ESP32 via `idf.py monitor`. Press **Ctrl+]** to exit.

---

### `scripts/stop.sh` — Stop emulation or monitor processes

```
./scripts/stop.sh
```

Kills any running `qemu-system-xtensa` or `idf_monitor` processes.

---

### `scripts/lint.sh` — Format and static-analysis checks

```
./scripts/lint.sh          # check only; exits 1 on violations
./scripts/lint.sh --fix    # auto-format all sources in-place
```

Runs two stages:
1. **clang-format** — checks `components/`, `main/`, and `tests/` sources.
2. **clang-tidy** — builds the `native-lint` CMake preset and runs the tidy
   checks through the CMake compile-commands database.

Always run `./scripts/lint.sh` before committing. If it fails, run with `--fix`
to auto-correct formatting, then re-run to confirm clang-tidy passes.

---

### `scripts/test.sh` — Build and run unit tests

```
./scripts/test.sh
```

Four steps:
1. `conan install` — resolves dependencies (Catch2) into `tests/build/debug/`.
2. `cmake --preset native-debug` — configures the native test build.
3. `cmake --build --preset native-debug` — compiles the tests.
4. `ctest --preset native-debug --output-on-failure` — runs all tests.

Tests live in `tests/` and use the Catch2 framework. The test binary is a
host-native executable; no ESP32 hardware or QEMU is required.

---

## Typical Workflows

### First-time setup
```bash
./scripts/setup.sh --install
```

### Build and run in emulation
```bash
./scripts/build.sh
./scripts/emulate.sh     # Ctrl+] to quit
```

### Flash to hardware
```bash
./scripts/flash.sh
./scripts/monitor.sh     # Ctrl+] to exit
```

### Run tests
```bash
./scripts/test.sh
```

### Lint before committing
```bash
./scripts/lint.sh --fix  # auto-fix formatting
./scripts/lint.sh        # confirm clean
```

## Conventions for Agents

- **Do not modify `build/` or `tests/build/`** — these are generated outputs.
- **Do not commit `tests/CMakeUserPresets.json`** — it is auto-generated by
  Conan and listed in `.gitignore`.
- **Adding a new component**: create `components/<name>/CMakeLists.txt`,
  `components/<name>/<name>.cpp`, and `components/<name>/include/<name>.h`.
  Follow the clang-format style enforced by `lint.sh`.
- **Adding a new test**: add test cases in `tests/test_helloworld.cpp` or
  create a new `tests/test_<name>.cpp` file and register it in
  `tests/CMakeLists.txt`.
- **Run `./scripts/lint.sh` after every source change** and fix all reported
  issues before considering a task complete.
- **Run `./scripts/test.sh` after every logic change** to verify nothing is
  broken.
- **Serial port**: use the `STEMBOT_PORT` environment variable to override the
  default port when flashing or monitoring.
- **ESP-IDF version**: `esp-idf` is installed at `~/esp/esp-idf`. The scripts
  source `export.sh` automatically — do not require the user to pre-source it.
